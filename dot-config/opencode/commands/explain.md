description: Deep dive explanation of code
---

Provide a thorough explanation of the code in `$ARGUMENTS`.

Structure the explanation as:
1. **Purpose**: What does this code do at a high level?
2. **How it works**: Walk through the logic step by step
3. **Key design decisions**: Why was it written this way? What tradeoffs were made?
4. **Dependencies**: What does it depend on? What depends on it?
5. **Edge cases**: What edge cases does it handle (or miss)?

For C code, also cover:
- Memory management patterns (who allocates, who frees)
- Pointer ownership and lifetime
- Thread safety considerations

For Python code, also cover:
- Type annotations and their accuracy
- Exception handling strategy
- Module/package structure

Keep the explanation clear and concise. Use code references with file:line format.
